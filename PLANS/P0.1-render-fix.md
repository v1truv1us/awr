# P0.1 Fix: HN Real-Site Rendering — Implementation Plan

## Problem Statement

`awr https://news.ycombinator.com --width 80` produces output where content
runs together with horizontal-rule-style dividers that exceed 80 characters.
Root causes:

1. **`renderTable` separator alignment**: The row-separator dash count
   (`widths[ci] + 2` at line 823) is based on measured widths before clamping.
   If a measured column exceeds `per_col_width`, the separator dashes misalign
   with the clamped cell content, and the total line can exceed `max_width`.

2. **Link-density not applied to table render mode**: `chooseContentRoot` in
   `browse_heuristics.zig` correctly identifies high-link-density tables as
   link-list containers (scoring +3 for `<td>`, +1.5 for `<tbody>`), but
   `renderTable` always renders as aligned columns with separators. HN-style
   tables (title-link row + metadata row × N) should render as vertical lists
   without column separators.

3. **Missing test coverage**: The existing HN fixture at `render.zig:1347`
   asserts line-length bounds but does not verify that link-list tables skip
   separator generation.

## Change 1: Clamp column widths before separator computation

**File**: `src/render.zig`

**Location**: `renderTable`, lines 794–817

**Current code** (lines 794–817):
```zig
const per_col_width = tableColumnWidthLimit(state.opts.max_width, max_cols);

// Measure column widths.
var widths = try state.allocator.alloc(usize, max_cols);
defer state.allocator.free(widths);
@memset(widths, 0);
var preview_link_index = state.links.items.len;
for (rows.items) |row| {
    var ci: usize = 0;
    for (row.children.items) |cell| {
        if (cell == .element and isCellTag(cell.element.tag)) {
            if (ci < max_cols) {
                if (collectTableCellDisplayText(state, cell.element, &preview_link_index)) |text| {
                    defer state.allocator.free(text);
                    widths[ci] = @max(widths[ci], measureWrappedWidth(text, per_col_width));
                } else |_| {}
            }
            ci += 1;
        }
    }
}
for (widths) |*width| {
    width.* = @max(@min(width.*, per_col_width), 1);
}
```

**Issue**: The clamping loop (lines 815–817) runs after measurement, so the
separator logic at line 825 already sees clamped values. This is correct in
the current flow. **No structural change needed here** — the widths are
already clamped before the row loop.

**However**: The `measureWrappedWidth(text, per_col_width)` at line 808 is
correct (it wraps at `per_col_width`), and the clamp at lines 815–817 caps
at `per_col_width`. The separator at line 825 (`widths[ci] + 2`) therefore
uses clamped widths. The real gap is the link-list detection (Change 2).

## Change 2: Link-density heuristic → list-style table rendering

**File**: `src/render.zig`

**Location**: New private function + `renderTable` entry

### 2a. Add `isLinkListTable` helper (new function, before `renderTable`)

Insert before line 774:

```zig
fn isLinkListTable(elem: *const dom.Element) bool {
    var link_count: usize = 0;
    var text_bytes: usize = 0;
    var paragraph_count: usize = 0;
    countTableLinkMetrics(elem, &link_count, &text_bytes, &paragraph_count);
    if (link_count < 6) return false;
    if (text_bytes < 80) return false;
    if (paragraph_count > 0) return false;
    const density = if (text_bytes == 0) 0.0 else @as(f64, @floatFromInt(link_count)) / @as(f64, @floatFromInt(text_bytes));
    return density >= 0.35;
}

fn countTableLinkMetrics(elem: *const dom.Element, link_count: *usize, text_bytes: *usize, paragraph_count: *usize) void {
    for (elem.children.items) |child| {
        switch (child) {
            .text => |text_node| {
                for (text_node.data) |ch| {
                    if (!std.ascii.isWhitespace(ch)) text_bytes.* += 1;
                }
            },
            .element => |child_elem| {
                if (eql(child_elem.tag, "a")) link_count.* += 1;
                if (eql(child_elem.tag, "p")) paragraph_count.* += 1;
                countTableLinkMetrics(child_elem, link_count, text_bytes, paragraph_count);
            },
            else => {},
        }
    }
}
```

**Logic**:
- Threshold: ≥6 links, ≥80 non-whitespace bytes, 0 paragraphs
- Link density ≥ 0.35 (link chars / total text chars)
- Mirrors the `scoreLinkListContainer` scoring from `browse_heuristics.zig:150`
  but simplified for runtime decision in the renderer

### 2b. Conditionally skip separators in `renderTable`

**Location**: `renderTable`, line 821–828

**Current code**:
```zig
// Separator between rows.
if (ri > 0) {
    for (0..max_cols) |ci| {
        if (ci > 0) try state.writeByte(w, ' ');
        for (0..widths[ci] + 2) |_| try state.writeByte(w, '-');
    }
    try state.newline(w);
}
```

**Change**: Add `is_link_list` flag computed once at function entry, then:

```zig
const is_link_list = isLinkListTable(elem);

// Render each row.
for (rows.items, 0..) |row, ri| {
    if (ri > 0 and !is_link_list) {
        for (0..max_cols) |ci| {
            if (ci > 0) try state.writeByte(w, ' ');
            for (0..widths[ci] + 2) |_| try state.writeByte(w, '-');
        }
        try state.newline(w);
    }
    // ... rest of row rendering unchanged
}
```

**Rationale**: HN's inner stories `<table>` (6+ links, 0 `<p>` tags, title text
~50+ bytes per row) passes the `isLinkListTable` check, so separators are
skipped. Regular data tables (with headers, metadata `<p>` tags, low link
density) continue to render with aligned columns and separators.

## Change 3: Enhance test fixture for HN-like table layout

**File**: `src/render.zig`

**Location**: Test section, after existing HN test (after line 1376)

**New test** — asserts that link-list tables (high link density) do NOT emit
separator dashes:

```zig
test "render — link-list table (high link density) omits row separators" {
    var doc = try dom.parseDocument(std.testing.allocator,
        \\<html><body><table><tr><td>
        \\  <table>
        \\    <tr><td><a href="https://example.com/1">Alpha story with a long title that should wrap cleanly</a></td></tr>
        \\    <tr><td>42 points by alice 2 hours ago | <a href="item?id=1">12 comments</a></td></tr>
        \\    <tr><td><a href="https://example.com/2">Beta story with a long title that should stay discrete and readable</a></td></tr>
        \\    <tr><td>21 points by bob 1 hour ago | <a href="item?id=2">8 comments</a></td></tr>
        \\  </table>
        \\</td></tr></table></body></html>
    );
    defer doc.deinit();

    var model = try renderBrowseModel(std.testing.allocator, &doc, .{
        .ansi_colors = false,
        .max_width = 40,
    });
    defer model.deinit();

    // Key assertion: no separator dashes in link-list table output
    try std.testing.expect(std.mem.indexOf(u8, model.text, "----") == null);

    // Content must still be present
    try std.testing.expect(std.mem.indexOf(u8, model.text, "Alpha story") != null);
    try std.testing.expect(std.mem.indexOf(u8, model.text, "Beta story") != null);

    // Line width enforcement
    for (model.lines) |line| {
        const text = model.text[line.start..line.end];
        if (text.len > 0) try std.testing.expect(text.len <= 40);
    }
}

test "render — regular table (low link density) keeps row separators" {
    var doc = try dom.parseDocument(std.testing.allocator,
        \\<html><body><table>
        \\  <tr><th>Name</th><th>Description</th></tr>
        \\  <tr><td>Alice</td><td>Works on compilers.</td></tr>
        \\  <tr><td>Bob</td><td>Works on networks.</td></tr>
        \\</table></body></html>
    );
    defer doc.deinit();

    var model = try renderBrowseModel(std.testing.allocator, &doc, .{
        .ansi_colors = false,
        .max_width = 60,
    });
    defer model.deinit();

    // Regular tables (low link density) should have separator dashes
    try std.testing.expect(std.mem.indexOf(u8, model.text, "---") != null);

    // Content present
    try std.testing.expect(std.mem.indexOf(u8, model.text, "Alice") != null);
    try std.testing.expect(std.mem.indexOf(u8, model.text, "Bob") != null);
}
```

**Rationale**:
- First test verifies link-list tables (HN-style) skip separators
- Second test verifies regular data tables still get separators
- Both assert line-width bounds to catch regressions

heuristic (`isLinkListTable`) is simpler and self-contained, avoiding coupling
between rendering and content-heuristics modules.
